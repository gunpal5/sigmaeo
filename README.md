## How to Submit an Issue

### Bug Reports

When reporting a bug, please include:

1. **Summary**: Brief description of the issue
2. **Steps to Reproduce**: Detailed steps to replicate the problem
3. **Expected Behavior**: What you expected to happen
4. **Actual Behavior**: What actually happened
5. **Environment**: Browser, OS, account tier
6. **Screenshots**: If applicable

**Use the `bug` label for bug reports.**

### Feature Requests

When requesting a feature, please include:

1. **Problem Statement**: What problem does this solve?
2. **Proposed Solution**: How should this feature work?
3. **Use Case**: Who benefits and how?
4. **Alternatives**: Other solutions you've considered

**Use the `enhancement` label for feature requests.**
